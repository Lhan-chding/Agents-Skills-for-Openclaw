# Delivery Checklist

Use this checklist before final delivery.

- [ ] Normalize the task and restate the problem clearly.
- [ ] Extract domain and technical keywords.
- [ ] Classify security sensitivity and note sensitive surfaces.
- [ ] Inspect at least 10 genuinely relevant external sources when access is available, or document why that was not possible.
- [ ] Fill the evidence ledger only with sources that were actually opened.
- [ ] Complete the architecture brief before writing code.
- [ ] Define the file tree before code.
- [ ] State a responsibility for each planned file or module.
- [ ] Keep configuration separate from business logic.
- [ ] Include tests when applicable, or explain why tests are not applicable.
- [ ] Include minimal documentation and run instructions.
- [ ] Validate inputs and choose safe defaults.
- [ ] Avoid hard-coded secrets and use placeholders or secure injection patterns.
- [ ] Record dependency choices and any visible supply-chain risks.
- [ ] Review commands for privilege, scope, and safer alternatives.
- [ ] Require human confirmation for destructive, privileged, or infrastructure-changing actions.
- [ ] Run the lightweight security review and record findings.
- [ ] Distinguish verified findings from assumptions and remaining unknowns.
